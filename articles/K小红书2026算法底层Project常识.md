## SECTION 1:三条核心公理(每次写小红书前必读)

```
公理 1:算法主语变了
  过去:让用户"多停留"
  现在:让用户"心智停留 + 解决问题"

公理 2:搜索权重 ≥ 推荐权重
  决策型/知识型垂类:搜索是主战场
  K 的所有内容必须"被搜到 > 被刷到"
  搜索流量占比 30-65%,决策垂类是绝对主流

公理 3:深度长文是 2026 核心资产
  不是劣势,是利好
  800-1500 字甜区,算法奖励结构清晰的长文
```

## SECTION 2:2026 算法权重公式(记住关键数字)

### 推荐得分("星云 5.0")

```
推荐得分 = 基础质量分 × 0.3
        + 用户互动分 × 0.4  ← 含深度阅读率、收藏转化
        + 商业价值分 × 0.2
        + 社交关系分 × 0.1
        − 违规扣分
```

### CES 显性公式(仍保留)

```
CES = 点赞 × 1 + 收藏 × 1 + 评论 × 4 + 转发 × 4 + 关注 × 8

2026 变化:
  - 纯点赞权重下降
  - 长评论(>10 字)加权
  - 截图、保存图片被单列高价值行为
  - 深度阅读时长进入用户互动分
```

### 搜索排序权重

```
内容质量分: 35%  (信息完整、结构清晰、可信度)
用户互动分: 30%  (收藏 > 评论 > 点赞,3:2:1)
账号权威分: 25%  (垂直度、历史质量、无违规)
新鲜度:     10%
```

### 关键词布局权重

```
标题(前 10 字权重最高,占比 > 60%): 权重 4
正文前 100 字:                      权重 3
正文其余:                            权重 2
话题标签:                            权重 1
```

## SECTION 3:硬性阈值(每篇笔记必须达到)

### 阅读权重

```
有效阅读:
  图文停留 ≥ 25 秒
  视频观看 ≥ 70 秒

全域推荐硬性条件(进大流量池):
  图文平均停留 > 48 秒
  视频完播率 > 75%
```

### 流量池晋级

```
初始流量池 → 中级:
  CTR:图文 ≥ 2.5-3%,视频 ≥ 4%
  互动率 ≥ 5%
  完播/完读率 > 40-45%

中级 → 高级:
  收藏率 ≥ 3%
  评论质量高(长评论、关键词丰富)
```

### 三阶流量测试

```
第 1 阶:100-500 曝光(初始测试池)
第 2 阶:5000-2 万曝光(中级流量池)
第 3 阶:10 万 - 500 万+(高级/现象级)
```

### 账号标签形成期

```
前 7-15 篇:账号标签开始形成
前 15-30 篇:账号标签基本固化
30 篇后:转型有阵痛期
```

## SECTION 4:每篇笔记必守 5 条硬规则

```
规则 1:标题前 10 字 = 核心关键词 + 反常识钩子
  ✅ "县城做题家的三重劣势:2026 年第一次翻盘"
  ❌ "一个被领导压的下午,我突然想通了"(关键词在末尾)

规则 2:正文前 100 字必须做三件事
  a) 自然融入 2-3 次核心关键词
  b) 告诉用户"这篇解决什么问题"
  c) 建立第一个钩子

规则 3:字数落在 800-1500 甜区
  < 600:信息密度不够,搜索吃亏
  800-1500:大多数主题最优
  > 2000:完读率风险,必须严密结构

规则 4:每 200-300 字一个节奏点
  小标题 / 金句 / 数字 / 对话
  目的:守住停留 > 48 秒的硬线

规则 5:结尾引导"收藏 > 评论 > 点赞"
  "这篇值得收藏" ✓ (权重最高)
  "评论区聊聊" ✓ (权重第二)
  "点个赞" ✗ (权重最低,不提)
```

## SECTION 6:K 的体制内话题包装规则(专属)

```
禁用词(平台可能人工复核):
  国企 / 央企 / 公务员 / 电力 / 政府 / 机关 /
  党政 / 基层干部 / 迎检 / 台账 / 议案 / 决议 /
  编制 / 事业单位

慎用词:
  体制内(偶尔用可以,不要频繁)

禁止表达:
  - 当前领导班子、最近政策、新上任的某某
  - 政策评价、政治倾向
  - 具体地名、部门名、职位名
  - 个人遭遇的直接暴露

安全替代:
  - "某组织""某公司""某单位"
  - 群像视角("我见过的兄弟们")
  - "中层管理""基层管理"
  - "小镇做题家""县城做题家"

K 的铁律:
  所有体制内话题必须用群像视角,
  不暴露个人遭遇的具体时间/事件/人物。
```

## SECTION 7:限流红线(2026 史上最严)

### AI 内容红线

```
合规(必须做):
  ✅ 内容中声明"AI 共创"或"AI 辅助创作"
  ✅ 必须有 K 的人格指纹(具体经验、第一人称、真实数字)
  ✅ Claude 负责打磨文笔,K 负责真实洞察

违规(直接封号):
  ✗ 全账号 AI 托管、无人工筛选
  ✗ AI 冒充名人、虚构身份
  ✗ AI 批量水文、养号
  ✗ AI 造假测评、伪造数据
```

### 站外导流红线

```
✗ 微信号、VX 谐音、二维码、交流群
✗ 跳转其他平台交易链接
✓ 引流走小红书站内私信
```

### 极限词(直接限流)

```
"国家级、世界级、最佳、唯一、首个、首选"
"顶级、TOP1、第一、永久、独家"
"100% 有效、0 风险、包过、稳赚"
"根治、治愈、7 天见效、无副作用"
```

### 矩阵号严打

```
判定:
  文案重复率 > 70% → 限流
  同 IP/设备多号 + 同质 → 封号

K 的两个号必守:
  风格、人设、内容结构必须显著差异化
  不互评、不互赞、最好不同设备
```

### 信用分

```
保持 > 90
单日删除笔记 > 3 篇 → 扣分
发布前违禁词检测,不要发出后再删
```

## SECTION 9:AI 协作时必须遵守的 10 条

当 AI 帮 K 写小红书内容时,必须:

```
1. 标题前 10 字放核心关键词 + 反常识钩子
2. 正文前 100 字融入 2-3 次关键词 + 告诉读者这篇讲什么
3. 字数落在 800-1500(决策类可到 2000)
4. 每 200-300 字一个节奏点(小标题 / 金句 / 数字)
5. 结尾引导"收藏 > 评论",不提"点赞"
6. 体制内话题严格用群像视角,不暴露具体身份
7. 涉及 AI 辅助的内容必须在文末声明"K 的洞察 + AI 共创"
8. 标签给 5-8 个(核心词 + 场景词 + 长尾词)
9. 判断这篇发哪个号:
   - 生活派视角 / 做题家共鸣 → K 写字的地方
   - 方法论 / AI 实验 / K-LAB 编号 → K 的赛博实验室
10. 发布前提醒 K 用违禁词检测工具自查
```
