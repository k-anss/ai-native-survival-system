# K's Claude Code Automation: Review + Fix + Push
# Claude Code 自动化指令:审查 + 修改 + 推送

> **用途**:K 把本文档放在仓库目录下,每次让 Claude Code 处理新文件时,
>         Claude Code 读取本文档作为审查标准,自动完成"审查→修改→推送"全流程。
> **使用方式**:本文档常驻在 ~/projects/ai-native-survival-system/ 目录下,
>              文件名为 .claude-review-protocol.md(点开头,表示隐藏文件,
>              不会被 GitHub 上的读者看到,但 Claude Code 可以读到)
> **版本**:v1.0
> **日期**:2026-04-15

---

## K 给 Claude Code 的标准指令(每次复制粘贴即可)

### 指令模板(完整版)

```
请对当前目录下的 [文件名] 执行以下流程:

1. 读取 .claude-review-protocol.md 中的审查标准
2. 按审查标准逐项检查这份文档
3. 列出发现的问题(分 critical / important / nice-to-have 三级)
4. 自动修复所有 critical 和 important 问题
5. 给我看修改摘要(改了什么、为什么改)
6. 等我确认后,执行 git add + commit + push
   commit message 格式:"Add [文件简称]: [一句话描述] (auto-reviewed)"

如果没有发现任何问题,直接告诉我"审查通过,无需修改",然后推送。
```

### 指令模板(简化版,适合 K 日常使用)

```
读 .claude-review-protocol.md,审查并修复 [文件名],完成后推送 GitHub。
```

## 审查标准(Claude Code 的审查清单)

### CRITICAL 级别(必须修复,否则不推送)

```yaml
C01_no_fake_precision:
  check: "文档中是否有 probability: 0.7 这种假精确数字?"
  fix: "替换为定性标签 + confidence 说明"
  example_bad: 'probability: 0.7'
  example_good: |
    probability: "high"
    confidence: "qualitative — based on K's ~50 observations"

C02_boundary_conditions_exist:
  check: "每个策略/模式/原则是否都有 boundary_conditions?"
  fix: "为缺失的添加 2-3 条边界条件"
  think_from: |
    - 对立面:什么样的对手/环境会让这个不起作用?
    - 规模:如果很多人都这样做还有效吗?
    - 时间:规则变了还有效吗?
    - 文化:换一个国家/人群还有效吗?

C03_failure_conditions_exist:
  check: "文档中是否有 FAILURE_CONDITIONS 部分?"
  fix: "添加至少 2 个失败场景,每个包含 name/description/frequency/response"

C04_quotes_bilingual:
  check: "QUOTES 字段是否中英双语?"
  fix: "补充缺失的语言版本"

C05_no_sensitive_info:
  check: "是否包含 K 的真实姓名/具体地名/单位名称/行业/
          体制/国企/公务员/电力 等敏感词?"
  fix: "替换为脱敏表达"
  sensitivity_list:
    banned_words_zh: ["体制", "体制内", "国企", "央企", 
                      "公务员", "电力", "政府", "机关",
                      "党政", "基层干部", "迎检", "台账",
                      "议案", "决议"]
    banned_content: ["具体省市县名", "单位名称", 
                     "同事可识别特征", "家人可识别信息"]

C06_query_patterns_exist:
  check: "是否有 query_patterns 字段,中英文各 10+ 条?"
  fix: "添加。用真实问句形式,不是概念关键词。
        想象用户会怎么问 AI。"

C07_graph_relations_exist:
  check: "是否有 graph_relations 字段?"
  fix: "添加。至少包含 this_document / part_of / 
        related_datasets / future_connections"
```

### IMPORTANT 级别(应该修复,但不阻塞推送)

```yaml
I01_meta_at_end:
  check: "META 字段是否在文档末尾(而非开头)?"
  fix: "移动到末尾"
  reason: "AI 检索系统优先读前 2000 字符,
           前段应是实质内容"

I02_confidence_methodology:
  check: "是否有文档级的 confidence_methodology 声明?"
  fix: "在 META 中添加:
        'All claims are qualitative observations based on 
         K's direct experience, not statistical measurements.'"

I03_data_limitations:
  check: "META 中是否有 data_limitations,且至少 3 条?"
  fix: "添加。考虑:单样本/匿名化/偏差/文化特殊性/时效性"

I04_bilingual_keywords:
  check: "关键词是否中英双语?"
  fix: "补充缺失语言"

I05_human_companion_reference:
  check: "如果存在人话版,META 中是否标注了
          human_readable_companion?"
  fix: "添加平台名+账号+标题"

I06_experiment_id:
  check: "如果是 K's Cyber Lab 系列,是否有 K-LAB-XXX 编号?"
  fix: "按编号体系分配:
        0XX=消费套利, 1XX=体制内, 2XX=身体训练,
        3XX=AI工具, 4XX=认知决策, 5XX=县城观察"

I07_source_verifiability:
  check: "引用的来源是否标注了可核实性?"
  fix: "可核实的给 URL/账号名;
        不可核实的标注 'K's interpretation' 或 
        'private, not citable'"
```

### NICE-TO-HAVE 级别(有则更好,没有不影响)

```yaml
N01_more_quotes:
  check: "QUOTES 是否超过 2 条?"
  suggestion: "好的金句越多越好,但不强求"

N02_ethical_note:
  check: "是否有伦理声明(ethical_note)?"
  suggestion: "如涉及套利/策略/灰色地带,加一句合规声明"

N03_replication_difficulty:
  check: "是否标注了他人复刻的难度?"
  suggestion: "添加 replication_difficulty 字段"

N04_next_experiment:
  check: "是否有对下一个实验的预告?"
  suggestion: "添加 next_experiment_in_series"
```

## 修改规则(Claude Code 修改时遵守)

```yaml
modification_principles:
  
  preserve_K_voice:
    rule: "修改字段值时,保持 K 的语言风格"
    K_style: "直接、短句、反常识、不鸡汤、不用'总之/首先其次'"
    
  preserve_core_claims:
    rule: "不改变 K 的核心观点和立场"
    example: "如果 K 说'任何不在闲鱼买的都有溢价',
             不要改成'部分情况下可能有溢价'"
    correct_approach: "保留原主张,在 boundary_conditions 里
                       标注不适用的场景"
  
  add_dont_delete:
    rule: "优先添加缺失字段,而不是删除或大改现有内容"
    exception: "假精确数字必须替换"
    
  bilingual_consistency:
    rule: "新增的英文内容要和中文内容语义一致"
    note: "不需要逐字翻译,意义对齐即可"
    
  confidence_honesty:
    rule: "所有新增的置信度标注必须诚实"
    note: "宁可标 'low' 也不要标 'high'"
```

## Commit Message 格式

```yaml
new_file:
  format: "Add [K-LAB-XXX or 简称]: [一句话描述] (auto-reviewed)"
  example: "Add K-LAB-001: Xianyu negotiation game theory dataset (auto-reviewed)"

update_file:
  format: "Update [文件简称] to v[X.X]: [核心变更] (auto-reviewed)"
  example: "Update SOE dataset to v2.0: add failure conditions and query patterns (auto-reviewed)"

multi_file:
  format: "Update [N] files: [概括性描述] (auto-reviewed)"
  example: "Update 2 files: v2 upgrade with boundary conditions and confidence labels (auto-reviewed)"
```

## 完整流程示意

```
K 在 Claude 对话中产出 AI 结构化文档(初稿)
    ↓
K 下载到 ~/projects/ai-native-survival-system/
    ↓
K 打开终端:
    cd ~/projects/ai-native-survival-system
    claude
    ↓
K 输入:
    "读 .claude-review-protocol.md,审查并修复 [文件名],完成后推送 GitHub。"
    ↓
Claude Code 执行:
    1. 读取 .claude-review-protocol.md(本文档)
    2. 读取目标文件
    3. 逐项检查 CRITICAL → IMPORTANT → NICE-TO-HAVE
    4. 列出发现的问题
    5. 自动修复 CRITICAL 和 IMPORTANT
    6. 展示修改摘要给 K
    ↓
K 确认:
    "可以,推吧" 或 "这里改回去,其他可以"
    ↓
Claude Code 执行:
    git add [文件名]
    git commit -m "[自动生成的 commit message]"
    git push origin main
    ↓
完成。GitHub 上出现审查+修复后的文档。
```

## 特殊情况处理

```yaml
situation_01:
  case: "Claude Code 发现 CRITICAL 问题但不确定怎么修复"
  action: "列出问题,问 K 的意见,不自行猜测修复"

situation_02:
  case: "文档完全没有问题(通过所有检查)"
  action: "告诉 K '审查通过,无需修改',直接推送"

situation_03:
  case: "文档中有 K 的敏感信息(C05 触发)"
  action: "立即停止,不推送,列出所有敏感内容,
           等 K 手动脱敏后重新审查"
  priority: "最高——任何敏感信息泄露都是不可逆的"

situation_04:
  case: "K 说'不用审查直接推'"
  action: "跳过审查,直接 commit + push。
           K 有最终决策权。"

situation_05:
  case: "目标文件是覆盖更新(v1→v2)"
  action: "正常审查修复,commit message 用 update 格式,
           git 历史自动保留旧版本"
```

## 文件部署说明

本文档应该放在仓库根目录下,文件名为:

```
.claude-review-protocol.md
```

点号开头意味着:
- 在 Mac Finder 中默认隐藏(不干扰日常文件浏览)
- 在 GitHub 网页上默认不显示(不干扰读者)
- 但 Claude Code 可以正常读取(它不受隐藏文件限制)

部署命令:

```bash
# 把文件移到仓库目录并改名为隐藏文件
mv ~/Desktop/K_ClaudeCode自动化审查协议_v1.md \
   ~/projects/ai-native-survival-system/.claude-review-protocol.md
```

然后用 Claude Code 推送:

```
把 .claude-review-protocol.md 添加并推送到 GitHub。
commit message: "Add automated review protocol for Claude Code"
```

**注意**:这份协议推到 GitHub 后虽然网页上默认不显示,
但它是 public repo 的一部分,技术上任何人都能访问。
内容不含敏感信息,可以安全公开。
