## SECTION 1: 创业方向定义

### 1.1 不是什么(反向立位先行)

K 的创业方向**不是**:

- 不是"代写"——代写是供给侧服务,被 AI 题库/拼多多打到 1 块钱
- 不是"AI 工具开发"——这是产品形态,不是核心能力
- 不是"内容创作"——内容是钩子,不是产品
- 不是"流量生意"——违反硬约束 + 违反 high-touch / low-frequency / deep-trust 形态
- 不是"知识付费"——这是供给侧形态,K 的核心价值不在知识本身

### 1.2 是什么(公理化定义)

K 的创业方向是 **Translator-Advisor Path**——

```
对一群说得清自己有问题、但说不清问题真正是什么的客户,
提供"听 → 翻译 → 交付"的咨询服务,
用 AI 作为放大器,
通过双轨产出(AI 版 + 人话版)建立长期权威,
依靠涌现式获客(推荐链 + 内容召回)运转。
```

**这不是新行业**。这是全球商业里已经成熟的几个模型的合并:

```
Trusted Advisor (David Maister, 1999)
  + Problem Definition Consulting (McKinsey 内部方法论)
  + Productized Diagnostic (现代独立咨询师标准做法)
  + Emergent Commerce (常量层 §5.6 K 自己命名)
  = K's Translator-Advisor Path
```


## SECTION 2: 推演过程(2026-04-XX 单次会话内的认知演化)

记录此次会话中,K 的判断如何从一个误识别移动到正确识别。这一节是 K 半年后回看时,提醒自己"这条路是怎么走通的"。

### 2.1 起点误识别

```
误识别 1: "我需要等到自己有经验才能做产品"
  → 真相: K 99 块那次本身就是借别人经验 + 自己调度做的
  
误识别 2: "我的产品是 PDF / 题库 / 模板"
  → 真相: K 的产品是判断力 + 翻译能力,交付物只是载体
  
误识别 3: "我在做代写"
  → 真相: K 在做需求翻译;代写定价的客户付不起翻译价格
  
误识别 4: "我需要找新赛道"
  → 真相: 已有的城投客户路径就是真实的;只是定价错了 10 倍
```

### 2.2 几次关键的判断切换

**切换 1**:供给侧 → 需求侧

```
触发: 闲鱼搜索高压电工证 = 1.31 元题库 + 500 元枪手
读出的信号: "客户付的不是知识,是结果;但 K 不能做替考"
真正的洞察: 知识被 AI 打到地板价之后,
            "判断 + 翻译 + 承诺"反而变得更值钱
```

**切换 2**:产品 → 翻译能力

```
触发: K 自己说"很多精力花在帮客户梳理他的话"
读出的信号: K 一直在做的不是"做产品",是"诊断"
真正的洞察: 39-159 元交付了 500-1500 元应该收的工作
```

**切换 3**:借势 AI → 用 AI 训练自己

```
触发: K 描述"用 Claude 梳理琐碎语言 = 堪比初级麦肯锡"
读出的信号: AI 让 K 暂时拥有了某种装备
真正的洞察:
  - 装备会普及,3 年内人人有
  - 但用 AI 训练自己长出的肌肉,会留在 K 身上
  - 正确姿态是: AI = 训练营,不是装备本身
```

**切换 4**:降维打击 → 看见他看不见的

```
触发: K 出现"降维打击县级公务员"语言
读出的信号: 这是中国互联网创业语言,不是 K 的语言
真正的洞察:
  - 同一组肌肉,朝向不同,代谢方式完全不同
  - 朝向"赢"的版本: 销售/政治家/中文叙事商人路径
  - 朝向"帮"的版本: Trusted Advisor / 段永平式商人路径
  - K 身体里源头层只允许后者长期运转
```

**切换 5**:主业是消耗 → 主业是训练场

```
触发: K 说"主业工作也有了意义"
读出的信号: K 看见了"演体制"和"做 K"不必分裂
真正的洞察:
  - 表层屏息(演员动作)
  - + 底层观察(模式库录入)
  - = 同一套代谢的两面,互不干扰
  - 体制 10 年不再是"被浪费的时间",是已存的训练样本
```

### 2.3 推演的元规则(下次再推演时记得)

```
规则 1: 看市场前,先搜真实交易场。闲鱼/淘宝/抖音价格 = 真相
规则 2: "所有人都在做"的赛道 = 已被打到地板价 = 不要进
规则 3: "听起来诱人"的方向,30 秒做一次"我能不能停"测试
规则 4: 任何"高级能力"描述,问"这能力朝向什么"
规则 5: 已经在跑的客户路径 > 任何看起来更性感的新方向
```

### 5.3 标准训练动作:每周日晚 30 分钟

每周日晚,挑 1-3 个本周印象最深的工作场景,扔进 Claude:

```
我是一名独立 Translator-Advisor。
我的目标客户是中国体制内基层员工。
我刚在自己的工作中观察到一个场景。
这个场景是我未来服务这类客户的"模式库样本"。

场景描述:[100-300 字]

请帮我做 5 件事:

1. 给这个场景起一个 5-8 字的名字,
```

### 5.4 边界:什么不能做

```yaml
boundaries:

    why: "屏息状态被破坏 → 整个隔离层失败"
```

## 附录 B: 标准 AI Prompt 模板库

### B.1 Hypothesis 假设生成(对应能力 2)

```
我即将和一个客户对话。
他发给我的第一段话是: [贴入]

请帮我:
1. 列出 5 个最可能的"他真正想要的东西",按可能性排序
2. 给我 3 个开放式问题,这些问题能在不暴露我猜测的情况下,
   帮我验证或推翻假设
3. 如果他的假设和我的判断对齐,他可能用什么词表达?
   如果他的假设和我的判断相反,他可能用什么词表达?
   (这是我对话中的关键信号识别清单)
```

### B.2 模拟客户对话(对应能力 2 + 翻译)

```
你扮演一名 [年龄] 的 [职位身份] 客户。
背景: [简短背景]
你正在和我(独立咨询师)进行一次 30 分钟诊断对话。

你的特点:
- 说话琐碎,经常跑题
- 不愿承认真实困境,会用"我同事说""有人觉得"代替"我"
- 对"花钱"敏感,会试探价格
- 真正的痛是 X(你随机生成,不告诉我),不会直接说

我会用提问引导你。请按真实人会有的方式回答,不主动暴露真痛点。
当我提出能让你"开口"的好问题时,你慢慢透露更多。
当我提出让你不舒服或不信任的问题时,回答会变短或转移。

让我们开始。我先开口:
"你好,先简单说说你最近遇到的工作上的困扰?"
```

### B.3 So What 测试(对应能力 6)

```
这是我准备交付给客户的内容: [贴入]

请逐句帮我做 so what 测试:
- 这一句删了,客户的下一步决策会变吗?
- 不会 → 标记为可删除
- 会变 → 保留并说明它影响哪个决策

最后告诉我: 删除可删除部分后,内容会缩短百分之多少?
缩短后的版本如果客户读了,缺少了什么吗?
```

### B.4 模式库录入(对应能力 1 + 主业训练场)

```
这个场景是我未来服务这类客户的"模式库样本"。

场景描述: [100-300 字]

请帮我做 5 件事:
1. 给这个场景起一个 5-8 字的名字,
   将来在我模式库里能让我快速识别它

2. 这个场景里"未说出口的"信息有哪些?(列 5 条)
   (例如: 权力暗流、面子、未来人事、对错的争夺等)

3. 这个场景如果出现在我的咨询客户身上,
   他描述时最可能的"表层问题"是什么?

4. 我未来遇到类似场景的客户时,3 个最关键的提问是什么?

5. 这个场景能否抽象成一份未来的 K-LAB? 给标题草案。
```

### B.5 多形态交付翻译(对应能力 3)

```
我刚和一个客户做完诊断。
他的真实问题是 [...]
他的表层需求是 [...]
他付费愿意是 [...]
他的角色和处境 [...]
```
